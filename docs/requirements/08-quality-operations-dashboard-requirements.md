# 08 质量经营与运营看板需求文档

版本：v0.1  
日期：2026-06-15  
对应规划：公司级质量经营看板、业务域看板、Agent 运营看板

## 1. 背景和目标

平台不仅服务单个项目，也要支撑 1000+ 人研发组织的一年数百个项目质量经营。质量经营看板的目标是让管理层、业务域负责人和质量运营团队看到 AI 参与变更后的效率、质量、风险和瓶颈。

## 2. 看板层级

| 看板 | 目标用户 | 核心问题 |
| --- | --- | --- |
| 公司级看板 | CTO、研发负责人、质量负责人 | AI 转型后整体效率和质量是否可控 |
| 业务域看板 | 业务域研发/测试负责人 | 哪些域风险高、逃逸多、门禁命中多 |
| 项目组合看板 | 项目负责人、PMO | 哪些项目阻断、延期、证据不完整 |
| Agent 运营看板 | 平台 Owner、Agent Owner | Agent 是否可用、可信、可持续优化 |

## 3. 核心指标

年度指标包括：

- L3/L4 项目证据完整率。
- AI 参与 PR 自检说明覆盖率。
- AI 参与 PR 门禁执行率。
- 核心链路需求规则卡确认率。
- 高风险 PR 必跑测试推荐覆盖率。
- 候选测试用例 Review 率。
- P0/P1 入库用例自动化绑定率。
- 发布前阻断项闭环率。
- 线上逃逸缺陷复盘反哺率。
- Agent 输出人工反馈率。

## 4. 公司级看板

公司级看板应展示：

- 项目总数。
- L1-L4 分布。
- AI 参与 PR 数量和占比。
- 门禁通过、阻断、豁免趋势。
- Change Failure Rate。
- Defect Escape Rate。
- Evidence Completeness。
- Lead Time。
- 重点风险项目列表。

## 5. 业务域看板

业务域看板应展示：

- 订单、优惠、库存、支付、履约、数据、算法等域的项目数量。
- 各域 L3/L4 占比。
- 缺陷逃逸趋势。
- PR 门禁命中原因 Top N。
- 自动化覆盖和用例资产增长。
- 业务域阻断项。

## 6. Agent 运营看板

Agent 运营看板应展示：

- Agent 调用量。
- 调用成功率。
- 输出 Schema 合规率。
- 建议采纳率。
- 人工修改率。
- 误阻断率。
- 漏检案例数。
- 最近版本和 Eval 结果。

## 7. 运营机制

系统应支持生成以下会议材料：

- 每周平台接入和阻断问题例会。
- 双周业务域质量会。
- 每月 AI 质量经营会。
- 每季度 Agent 权限、策略和 ROI 复盘。

## 8. 一期范围

一期必须支持：

- 项目组合状态汇总。
- 风险等级分布。
- 证据完整率。
- PR 门禁结果统计。
- 测试资产 Review 和入库统计。
- Agent 调用和人工反馈统计。

一期暂不强制实现：

- 自动生成管理层 PPT。
- 高级预测模型。
- 跨年度趋势分析。

## 9. 验收标准

- 看板能按业务域、项目、风险等级筛选。
- L3/L4 项目证据不完整时能被管理层看板识别。
- Agent 输出被采纳、修改、驳回必须能统计。
- 月度报告能输出 AI 参与变更的效率、质量和风险结论。
