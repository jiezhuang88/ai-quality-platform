# 07 证据中心与工具集成需求文档

版本：v0.1  
日期：2026-06-15  
对应规划：Evidence Center、工具 Adapter、发布准入报告

## 1. 背景和目标

当前团队已有 Sonar、单测覆盖率、接口自动化覆盖率、接口响应时长、遗留缺陷、测试用例通过率、测试数据构造平台、性能平台和混沌工程等工具和指标。证据中心的目标是把分散工具结果转成统一质量证据，用于门禁、发布准入和复盘。

## 2. 证据类型

| 类型 | 来源 |
| --- | --- |
| 需求证据 | PRD、规则卡、歧义点确认 |
| 代码证据 | PR、Diff、AI 使用说明、Review 记录 |
| 静态扫描证据 | Sonar、SAST、SCA |
| 测试证据 | 用例、自动化执行、接口测试、E2E |
| 数据证据 | 数据银行任务、测试数据集 |
| 性能证据 | 性能测试报告、响应时长、容量结论 |
| 混沌证据 | 混沌实验、故障注入、恢复结果 |
| UAT 证据 | 验收记录、业务确认 |
| 发布证据 | 灰度、回滚、监控、审批 |
| 复盘证据 | 缺陷、事故、根因、反哺项 |

## 3. 证据对象字段

证据对象应包含：

- evidence_id。
- project_id。
- stage。
- evidence_type。
- source_system。
- source_url。
- status。
- result。
- summary。
- owner。
- created_at。
- updated_at。
- expires_at。
- related_requirement_ids。
- related_pr_ids。
- related_case_ids。
- risk_level。

## 4. 工具 Adapter

一期应支持以下 Adapter 的登记或模拟接入：

- Sonar Adapter。
- CI Adapter。
- 自动化框架 Adapter。
- 数据银行 Adapter。
- 性能平台 Adapter。
- 混沌工程 Adapter。
- 缺陷平台 Adapter。

每个 Adapter 必须定义：

- 认证方式。
- 数据拉取方式。
- 字段映射。
- 状态映射。
- 错误处理。
- 同步频率。

## 5. 证据完整率

系统应按项目风险等级计算证据完整率：

- 必需证据数量。
- 已完成证据数量。
- 失败证据数量。
- 过期证据数量。
- 阻断证据数量。

证据完整率不能只按数量计算，阻断类证据失败时应直接影响发布准入。

## 6. 发布准入报告

报告内容包括：

- 项目基本信息。
- 当前风险等级。
- 需求规则卡确认情况。
- PR 门禁结果。
- Sonar/覆盖率/CI 状态。
- 测试用例执行情况。
- 自动化、性能、混沌证据。
- UAT 结论。
- 遗留缺陷和豁免项。
- 阻断项。
- 发布建议。

## 7. 一期范围

一期必须支持：

- 统一登记证据。
- 手动录入或导入工具结果。
- 按项目和阶段查看证据。
- 生成发布准入报告。
- 识别缺失证据和阻断证据。

一期暂不强制实现：

- 所有工具实时 API 接入。
- 自动审批发布。
- 证据防篡改存证。

## 8. 验收标准

- L3/L4 项目能展示完整证据链。
- Sonar 阻断证据失败时发布准入不允许 pass。
- 发布准入报告能说明通过、阻断或有条件发布原因。
- 每条证据能追溯来源和关联对象。
