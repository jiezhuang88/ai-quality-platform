# 04 AI Coding PR 质量门禁需求文档

版本：v0.1  
日期：2026-06-15  
对应规划：AI Coding PR 门禁、多仓库 PR、Sonar/覆盖率/CI 证据

## 1. 背景和目标

AI Coding 会显著增加 PR 数量和代码改动规模。PR 阶段必须成为 AI 代码质量治理的核心入口，识别 AI 生成比例、变更影响面、跨仓库依赖、Sonar 问题、覆盖率下降和必跑测试缺口。

## 2. 业务流程

1. 在项目空间中登记或同步 PR。
2. PR 关联仓库、系统、需求规则卡和风险等级。
3. 开发填写 AI 使用说明和自检说明。
4. 平台采集 Sonar、CI、单测、覆盖率、SAST/SCA 结果。
5. PR Diff Agent 生成风险摘要和必跑测试建议。
6. 按 L1-L4 门禁规则判断是否通过。
7. 门禁结论进入证据中心。

## 3. PR 基础信息

PR 对象字段：

- PR ID。
- PR URL。
- 标题。
- 仓库。
- 分支。
- 作者。
- Reviewer。
- 关联需求规则卡。
- 关联系统。
- 变更文件列表。
- 风险等级。
- 状态。

## 4. AI 使用说明

AI 参与 PR 必须填写：

- 使用的 AI 工具。
- AI 使用范围。
- AI 生成比例。
- Prompt 假设或关键约束。
- 人工修改说明。
- 自检说明。
- 已补充的单测或自动化说明。

未填写 AI 使用说明的 PR 不允许进入 L2 及以上门禁通过。

## 5. 质量证据采集

一期需要支持登记或接入：

| 证据 | 要求 |
| --- | --- |
| Sonar | quality gate、bugs、vulnerabilities、code smells、coverage |
| CI | 构建状态、失败日志地址 |
| 单测 | 执行状态、覆盖率、失败用例 |
| 接口自动化 | suite、通过率、失败接口 |
| 覆盖率 | 当前覆盖率、基线覆盖率、差异 |
| SAST/SCA | 高危安全问题、依赖漏洞 |

## 6. PR Diff Agent 能力

PR Diff Agent 必须输出：

- 变更摘要。
- 影响系统和模块。
- 是否核心链路。
- 跨仓库风险。
- AI 高占比风险。
- Sonar/覆盖率/CI 风险。
- 必跑测试建议。
- Review 关注点。
- 阻断项建议。
- evidence_refs。
- human_checkpoints。

## 7. 门禁规则

| 等级 | PR 门禁 |
| --- | --- |
| L1 | PR Review、基础 CI、Sonar 无阻断 |
| L2 | L1 + 单测或接口自动化、需求规则确认 |
| L3 | L2 + 核心链路必跑测试、覆盖率不得低于基线、多系统影响说明 |
| L4 | L3 + 安全/架构/业务签核、回滚补偿方案、发布负责人确认 |

## 8. 一期范围

一期必须支持：

- 手动登记多仓库 PR。
- 登记 AI 使用说明。
- 登记 Sonar 和 CI 结果。
- 自动生成 PR 风险摘要。
- 输出门禁结论。

一期暂不强制实现：

- 自动拉取 Git diff。
- 自动创建 GitHub/GitLab Check。
- 自动阻断代码合并。

## 9. 验收标准

- 一个项目可关联多个仓库 PR。
- AI 参与 PR 缺少自检说明时门禁提示阻断。
- Sonar ERROR 不允许进入发布准入 pass。
- L3/L4 PR 能生成必跑测试建议。
