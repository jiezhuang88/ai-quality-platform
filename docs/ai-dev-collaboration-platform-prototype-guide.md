# AI 时代研发协同与质量工程平台原型说明

版本：v1.0  
原型位置：`web-quality-workbench/`

## 1. 原型目标

原型用于向老板和研发团队说明平台如何从“QA 质量工具”升级为“全研发协同平台”。

原型重点展示：

- 平台总览。
- 研发协同作业流。
- 风险分级和门禁。
- AI Agent 协同。
- 工具集成。
- 研发 cockpit。
- 变更报告导出。

## 2. 启动方式

```bash
cd web-quality-workbench
python3 server.py --host 127.0.0.1 --port 8787
```

访问：

```text
http://127.0.0.1:8787
```

## 3. 演示路径

### 3.1 管理层演示

1. 打开总览页，说明平台不是 QA 工具箱，而是全研发协同系统。
2. 打开研发协同页，说明从需求到复盘的端到端作业流。
3. 打开工具集成页，说明复用现有数据银行、自动化、性能、混沌、Sonar 等能力。
4. 打开 Agent 协同页，说明 AI 参与需求、PR、测试、失败归因和复盘。
5. 打开度量看板，说明 CTO 如何看到效率、质量、稳定性和 AI 代码风险。

### 3.2 研发团队演示

1. 点击“填充示例”。
2. 查看变更准入，说明研发需要填写业务目标、AI 使用说明、禁止范围、验收标准。
3. 查看风险分级，说明风险等级如何决定门禁强度。
4. 查看测试策略和安全供应链，说明缺哪些证据会阻断。
5. 查看报告页，说明平台如何输出 PR/发布证据。
