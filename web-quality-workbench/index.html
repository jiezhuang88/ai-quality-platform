平台定位

不是 QA 工具箱，而是面向产品、研发、测试、发布、业务验收的统一协同生产系统。

协同作业流

同一变更在平台内完成需求澄清、方案评审、PR 门禁、测试验证、UAT、发布和复盘
产品/业务

提供业务目标、验收标准、UAT 口径，确认全场券、商品券等业务规则。

研发

提交技术方案、AI Coding 使用说明、PR、单测、自检和回滚方案。

测试

编排数据银行、接口自动化、E2E、性能、混沌和探索测试证据。

发布/运维

确认灰度、监控、告警、回滚、发布窗口和异常响应预案。

AI Agent 协同

AI 负责分析、推荐和证据生成，人负责业务语义、风险裁决和例外审批

需求澄清 Agent

抽取业务规则、验收点、歧义和影响链路。

方案评审 Agent

识别架构、接口、数据一致性、回滚和依赖风险。

PR Diff Agent

分析 Diff、影响服务、历史缺陷和风险等级建议。

测试策略 Agent

推荐测试范围、数据需求、回归集和 UAT 关注点。

失败归因 Agent

分析 CI、自动化、日志失败，给出疑似根因。

复盘 Agent

把逃逸缺陷转化为新增用例、门禁规则和知识库条目。

工具集成

复用现有平台能力，由协同平台统一触发、收集证据和生成门禁结论
数据银行自动准备全场券、商品券、库存、支付异常等测试数据
自动化框架按风险推荐接口、E2E、回归测试集合
性能平台验证下单、结算、支付前校验等核心接口 P95/P99
混沌工程验证优惠、库存、支付依赖异常下的降级和恢复能力
Sonar/覆盖率提供静态扫描、复杂度、漏洞、单测覆盖率变化
监控/缺陷绑定发布监控，回收线上缺陷并反哺门禁

SDLC 全链路协同控制

从需求到运行复盘，每个阶段都有责任人、证据、AI 辅助和自动化门禁
需求准入 业务目标、验收标准、影响范围、风险评估
设计评审 接口契约、回滚策略、安全设计、数据恢复
AI 编码约束 AI 使用说明、Prompt 假设、禁止范围、人工复核
PR 准入 PR 描述、改动文件、测试说明、风险等级
测试验证 单元、集成、契约、E2E、安全、AI 测试分析
安全供应链 SAST、密钥扫描、SCA、SBOM、LLM 风险评审
发布控制 发布说明、回滚、监控、灰度、审批
运行复盘 质量指标、回归反馈、线上监控、缺陷复盘
